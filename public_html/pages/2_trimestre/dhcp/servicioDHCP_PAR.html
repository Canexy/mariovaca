Actividades sobre el servicio DHCP.

  1. ¿Para qué sirve el protocolo DHCP basado en este laboratorio?

El protocolo DHCP permite asignar automáticamente parámetros de red a los dispositivos que se conectan. En este laboratorio, se analiza cómo el servidor DHCP otorga direcciones IP de forma automática a los nodos de la red, lo que facilita tanto su configuración como su mantenimiento.

  1. ¿Cómo funciona este protocolo?

Cuando un dispositivo se une a la red, envía una solicitud al servidor DHCP para obtener su configuración de red. El servidor revisa su tabla y le asigna una dirección IP libre. A partir de entonces, el dispositivo utiliza la IP asignada para comunicarse dentro de la red.

  1. ¿Cómo se le asignarían direcciones estáticas desde el servidor DHCP a los clientes pc1 y pc2?

Se demostraría cambiando el modo de DHCP a estático. Hecho eso, se tiene que especificar en la tabla qué dirección se debe de asignar a cada nodo basado en su dirección MAC.

  1. ¿Por qué falla el uso de DHCP con pc3?

El error ocurre debido a la ausencia de los paquetes necesarios para que pc3 utilice DHCP. Estos paquetes, esenciales para la comunicación entre pc3 y el servidor DHCP, no están disponibles o no se están transmitiendo correctamente.

  1. ¿Cómo se puede solucionar el problema de pc3 de manera que también pueda configurarse automáticamente sin el uso de DHCP relay?

La solución consiste en configurar manualmente la red en pc3. Editando el archivo de configuración, se puede asignar una dirección IP estática, eliminando así la dependencia del servidor DHCP.

  1. ¿Qué otros parámetros se pueden configurar automáticamente con DHCP?

Además de la dirección IP, DHCP puede asignar automáticamente otros parámetros de red, como la máscara de subred, el servidor DNS, la puerta de enlace, la dirección DNS, el tiempo máximo de espera de ARP, la MTU y los servidores NIS, NTP, SMTP y TFTP. El servidor DHCP se encarga de proporcionar estos ajustes para facilitar la configuración de los dispositivos cliente.
