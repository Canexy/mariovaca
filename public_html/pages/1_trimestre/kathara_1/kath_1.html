Memoria Kathará Práctica #1

  1. Un único nodo (single host).

El comando usado para la creación de un único host (sin uso de un laboratorio) será kathara vstart, seguido de los argumentos -n ‘pc1’ que hará referencia al nombre que tendrá nuestra máquina. Al ejecutar dicho comando se habrá la terminal de nuestro host llamado pc1.

La máquina podrá se cerrará usando el comando kathara vclean -n ‘pc1’.

  1. Dos nodos en el mismo dominio de colisión A (en laboratorio).

La creación del laboratorio se hará en un nuevo directorio, donde se meterán todos los archivos referidos al mismo. Al ser de dos nodos, únicamente contendrá dos archivos con la configuración de los mismos bajo su mismo nombre (e.g pc1 y pc2) con la extensión .startup, y un archivo extra que hará de configuración para el laboratorio en sí, llamado lab.conf.

En su defecto, se creará además un directorio llamado shared.

Los archivos de cada nodo contendrán la información de red del mismo.

pc1.startup contendrá la siguiente línea de código:

ifconfig eth0 10.0.0.1

Y, en el caso de pc2.startup:

ifconfig eth0 10.0.0.2

Así, estaremos asignando una dirección de red específica a cada uno, con un mismo puerto Ethernet, eth0.

lab.conf, así como cada nodo, contendrá también información necesaria para la creación del laboratorio y su funcionamiento. Esta información relacionará cada nodo con su dominio de colisión.

Para el caso en concreto, lab.conf contendrá las siguientes líneas:

pc1[0]=A

pc2[0]=A

A hará referencia al dominio de colisión, dominio concretado por nosotros donde haremos que ambos nodos colisionen y se escuchen. 0 hará referencia al puente de red usado por el que se comunicarán los nodos.

Ahora, desde el propio directorio del laboratorio, abriremos una nueva terminal. Usando el comando kathara lstart lanzaremos el mismo.

En la práctica anterior usamos kathara vstart, ahora kathara lstart. Con el primero lanzamos una única máquina, con el segundo comando todo un laboratorio ya creado.

Ahora, lanzaremos el comando ping 10.0.0.1 desde la máquina pc2. Si obtiene respuesta, demostraremos que ambos nodos se pueden ver.

  1. Creación de un laboratorio de una red de 3 hosts conectados a un 'hub virtual'.

Para la creación de un laboratorio de 3 nodos, usaremos el laboratorio creado anteriormente. Únicamente modificaremos el archivo lab.conf, que tendrá las siguientes líneas:

pc1[0]=A

pc2[0]=A

pc3[0]=A

Y, además, crearemos un nuevo archivo llamado pc3.startup, que al igual que pc2 y pc1, tendrá la siguiente estructura:

ifconfig eth0 10.0.0.3

Así, cada nodo de nuestro laboratorio tendrá su propia dirección y mediante el archivo lab.conf, estarán conectados al mismo dominio de colisión, A.

Este será el resultado de lanzar el laboratorio, usando kathara lstart sobre una terminal en el directorio del mismo.

Ahora, haciendo un ping 10.0.0.2 desde pc1 a pc2, veremos lo que pc3 puede escuchar. Para ello, lanzaremos ese primer comando y luego usaremos un nuevo comando sobre pc3:

tcpdump -w /shared/captura.pcap

tcpdump es un comando que capturará el tráfico de red. Con -w y una ruta especificada, lo guardará donde le digamos con la extensión que queramos. Usaremos la extensión pcap para poder abrir dicha captura posteriormente con un programa llamado wireshark, usado para la escucha a tiempo real del tráfico de red, o en nuestro caso, para el análisis de un archivo que ya contiene dicho tráfico. Además, la ruta que usaremos será uno de los directorios que creamos anteriormente, shared.

Una vez hecho todo, ahora podemos abrir dicha captura (que estará en el directorio proporcionado) con wireshark.

Así, demostramos qué es lo que pc3 estaba escuchando cuando pc1 hacía ping a pc2. Como vemos, son todas las peticiones y respuestas entre los dos nodos. Se demuestra así, que estando sobre el mismo dominio de colisión, los 3 nodos son capaces de ver el tráfico de red entre ellas mismas, dando como resultado un pobre aislamiento y seguridad usando dicha topología.
